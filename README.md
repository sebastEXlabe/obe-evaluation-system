# OBE-CDIO 软件工程课程教学评价管理系统

> 基于成果导向教育(OBE)与CDIO工程教育模式的综合教学质量管理平台

**技术栈**: Spring Boot 3.4 + Vue 3 + PostgreSQL + MyBatis-Plus + JWT + DeepSeek AI | **测试**: 37/37 passed

---

## 一、项目概述

### 1.1 解决的核心痛点

| 痛点 | 解决方案 |
|------|---------|
| 小组协作"搭便车"问题 | 贝叶斯平滑贡献度算法 + Git/日志/问答多维数据 |
| 过程性考核数据难以采集 | Git自动采集 + AI问答记录 + 自测成绩 + 项目日志 |
| OBE达成度难以追踪 | CO→PO四级体系 + 加权计算 + 最小值法 |

### 1.2 核心理念

| 理念 | 系统体现 |
|------|---------|
| **OBE** 成果导向 | 课程目标→指标点→评价方式→达成度 四级体系 |
| **CDIO** 工程教育 | 构思-设计-实施-运行四阶段映射 |
| **AHP** 层次分析法 | 两两比较矩阵 + 一致性检验 CR<0.1 |
| **PDCA** 持续改进 | 达成度→改进建议→工单→追踪闭环 |

## 三、六大功能模块

### 模块一：课程目标与评价指标管理（OBE顶层设计）

**设计逻辑**: 先定义"要达成什么"，再设计"如何评价"，最后配置"谁来采集"——与"先考试再算分"相反。

**功能**:
- 三维度课程目标（知识/能力/素养）+ AHP权重校验
- 目标→指标点→评价方式→CDIO映射 四级分解
- CO→PO毕业要求映射矩阵（H/M/L三级支撑度）
- 权重实时可视化 + CR一致性检验

### 模块二：小组与岗位分工管理（CDIO组织架构）

**4人小组 + 四类岗位 + 岗位专项评价**:

| 岗位 | 评价维度 |
|------|---------|
| PM 项目经理 | 计划制定(35%) / 进度控制(35%) / 团队协调(30%) |
| DEVELOPER | 代码质量(40%) / 技术实现(35%) / 问题解决(25%) |
| TESTER | 测试覆盖(40%) / Bug跟踪(40%) / 质量报告(20%) |
| DOC_ADMIN | 文档规范(40%) / 完整性(35%) / 及时性(25%) |

### 模块三：MaxKB智能问答与学习行为采集（过程性数据入口）

- 7×24 AI智能助教（DeepSeek + 课程知识库）
- 自动关联知识点 + 提问记录 → 过程性评价数据
- AI自动出题批改 + 教师审核发布
- 教师学情看板：高频知识点热力图 + 活跃度预警
- 问答数据自动计入贡献度计算（权重25%）

### 模块四：项目过程追踪（CDIO实施支撑）

- 里程碑管理 + 任务看板（TODO/DOING/DONE）
- GitHub/Gitee自动采集Commit记录
- 项目日志 + 贡献记录 + 需求变更追踪

### 模块五：多元化过程性评价（考核体系）

**反搭便车贡献度公式**:
```
贡献分 = Git得分×0.35 + 日志得分×0.20 + (问答分+自测分)×0.25 + 贡献加分×0.20
个人最终成绩 = 小组总分 × 贡献系数(贝叶斯平滑) + 个人加分
```

### 模块六：达成度分析与可视化（OBE闭环）

- CO达成度加权计算（权重归一化）
- CO→PO毕业要求达成度（最小值法）
- 雷达图 + 趋势线 + 成绩回溯反查
- PDCA改进工单自动生成

## 四、数据闭环

```
AI问答→自测成绩→Git提交→项目日志
  → 过程性评价数据采集
  → 贡献度计算 → 个人成绩
  → 达成度分析 → 薄弱知识点识别
  → 改进建议生成 → 教师教学调整
  → 下一轮迭代
```

## 五、部署指南

### 启动服务
```bash
# PostgreSQL + Redis + MaxKB
docker-compose up -d

# 后端 (port 8989)
cd obe-server && mvn spring-boot:run

# 前端 (开发)
cd obe-web && npm run dev

# 前端 (生产)
cd obe-web && npm run build
cp -r dist/* ../obe-server/src/main/resources/static/
```
